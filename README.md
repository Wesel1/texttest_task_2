# Notification Deadline API

Небольшой сервис на FastAPI для хранения уведомлений сотрудников и расчёта дат напоминаний.

Сервис принимает событие сотрудника, считает дедлайн через 3 рабочих дня после даты события и возвращает даты напоминаний за 1, 3, 7, 14 и 30 рабочих дней до дедлайна. При расчётах учитываются выходные и праздники России.

## Стек

- FastAPI
- Pydantic
- holidays
- pytest

## API

### Создать уведомление

```http
POST /notifications
```

Пример тела запроса:

```json
{
  "employee_id": 10,
  "event_date": "2026-04-01",
  "event_type": "vacation"
}
```

Успешный ответ:

```json
{
  "status": "ok"
}
```

Если такое уведомление уже существует, сервис вернёт `409`.

### Получить напоминания сотрудника

```http
GET /notifications?employee_id=10
```

Пример ответа:

```json
{
  "reminders": [
    [
      "2026-04-03",
      "2026-04-01",
      "2026-03-26",
      "2026-03-17",
      "2026-02-20"
    ]
  ]
}
```

Если сотрудник не найден, сервис вернёт `404`.

## Тесты

В проекте написано 10 тестов:

- проверка преобразования строки в дату;
- расчёт дедлайна с пропуском выходных;
- расчёт дат напоминаний по рабочим дням;
- проверка дублей;
- сохранение нового уведомления;
- ошибка при повторном уведомлении;
- успешное создание уведомления через API;
- ошибка `409` при дубле через API;
- получение напоминаний через API;
- ошибка `404` для неизвестного сотрудника.

Запуск:

```bash
pytest
```
